# DX11 Multi-Window Sync Assistant v3.2

DX11 Multi-Window Sync Assistant 是一个 Windows 多窗口输入同步工具。它可以把一个主控窗口中的键盘和鼠标操作，同步转发到多个受控窗口，适合在同一台电脑上管理多个同类窗口。

v3.2 重点解决 Windows 10/窗口模式下的实际使用问题：键盘和鼠标同步可以单独开关，鼠标离开主控窗口客户区后不再误触发受控窗口，窗口列表刷新后按稳定规则排序，找主控窗口更直接。

---

## v3.2 更新

- **键盘/鼠标独立同步**：底部操作栏新增 `键盘: 开/关` 与 `鼠标: 开/关`，可以只同步键盘或只同步鼠标。
- **窗口模式鼠标保护**：鼠标事件只在光标位于主控窗口客户区内时转发；移到标题栏、边框、任务栏、桌面或其他窗口后停止转发。
- **稳定窗口排序**：窗口列表按进程名、窗口标题、PID、窗口句柄排序，避免依赖 Windows 枚举顺序导致刷新后位置随机。
- **v3.2 图标更新**：应用图标已替换为新的多尺寸 ICO 资源。
- **保留 v3.1 Win10 兼容修复**：继续嵌入 manifest、管理员权限、DPI 感知和 Win10/Win11 兼容声明。

---

## 功能特性

- **双列表选择**：左侧选择主控窗口，右侧勾选一个或多个受控窗口。
- **标题/进程过滤**：可按窗口标题或进程名快速筛选。
- **键盘同步**：转发 `WM_KEYDOWN` / `WM_KEYUP` / 系统按键消息。
- **鼠标同步**：转发鼠标移动、点击和滚轮，移动事件带节流保护。
- **热键黑名单**：可屏蔽 Alt+Tab、Win 键、功能键等不希望同步的按键。
- **Direct2D 自绘界面**：Win32 + Direct2D + DirectWrite，支持深色/浅色主题。
- **单文件运行**：Release 构建使用静态 CRT，无需额外 DLL 或运行时安装。

## 使用方法

1. 右键 `dx11_sync.exe`，选择 **以管理员身份运行**。
2. 在左侧列表选择主控窗口。
3. 在右侧列表勾选需要同步的受控窗口。
4. 按需要切换 `键盘: 开/关` 和 `鼠标: 开/关`。
5. 点击 `开始同步`。
6. 切换到主控窗口操作。
7. 需要停止时回到工具窗口点击 `终止同步`。

窗口模式下，鼠标同步只在主控窗口客户区内生效。鼠标移到任务栏或其他窗口时，受控窗口不会继续收到鼠标消息。

---

## 热键黑名单

点击 `热键黑名单` 可以配置不需要转发的按键。建议保留默认屏蔽项，尤其是：

- Win 键
- Alt+Tab
- Ctrl+Esc
- 可能导致切出窗口或系统操作的组合键

黑名单只影响键盘同步，不影响鼠标同步。

## 技术架构

| 模块 | 说明 |
|------|------|
| `src/app_window.*` | 主窗口、Direct2D UI、按钮命中、列表渲染、状态栏 |
| `src/sync_engine.*` | 键盘/鼠标低级钩子、同步开关、事件转发、鼠标客户区判断 |
| `src/window_manager.*` | 顶层窗口枚举、候选窗口过滤、稳定排序 |
| `src/key_blacklist.*` | 热键黑名单对话框与按键集合管理 |
| `resource.rc` | 图标与 manifest 资源嵌入 |
| `app.manifest` | 管理员权限、DPI 感知、Win10/Win11 兼容声明 |

核心流程：

```text
低级键鼠钩子
  -> 判断是否正在同步
  -> 判断主控窗口是否前台
  -> 鼠标额外判断是否在主控客户区
  -> 过滤黑名单按键
  -> PostMessageW 投递到受控窗口
```

---

## 兼容性与限制

- 需要管理员权限，否则全局低级钩子可能安装失败。
- 只转发 Windows 消息队列事件。部分游戏如果直接使用 Raw Input、DirectInput 或内核级反作弊输入路径，可能无法通过 `PostMessageW` 同步。
- 鼠标同步使用屏幕坐标转换到每个受控窗口客户区；不同窗口尺寸或 UI 布局不一致时，点击位置可能不完全对应。
- v3.2 不保存键盘/鼠标开关状态，启动后默认键盘和鼠标同步均为开启。
